# Логика и правила нард — пакет для старта проекта с нуля

Этот пакет содержит **готовый движок правил** (`nardy-engine.js`) и описание того,
**как устроена оригинальная игра** из распакованного APK. Цель — дать тебе чистую,
понятную основу, чтобы начать новый проект и сразу подгружать ассеты для сборки UI.

---

## 1. Честно: что можно и что нельзя «вытащить» из APK

Игра сделана на **Unity 6 (IL2CPP)**. Это значит, что код правил написан на C#,
затем скомпилирован в C++ и в нативный машинный код ARM64 (`lib/arm64-v8a/libil2cpp.so`,
118 МБ). **Читаемого исходного кода правил в файлах нет** — есть только процессорные
инструкции. Восстановить из них чистый C#/JS невозможно.

Что удалось извлечь из метаданных (`global-metadata.dat`) — это **структура**: имена
классов и пространств имён (3402 класса). Это показывает, КАК игра организована, но НЕ
реализацию методов.

**Главный вывод из структуры:** игра — **сетевой мультиплеер**, и авторитетные правила
живут на **сервере**. Клиент получает с сервера готовое `BoardState`, рисует его и шлёт
ходы. Поэтому полного движка правил в клиенте может не быть вовсе — он на их сервере,
к которому доступа нет.

Поэтому правильный путь (и он реализован здесь): **свой движок правил**. Нарды —
стандартная игра, правила открыты и не защищены авторским правом. `nardy-engine.js` —
это и есть та логика, протестированная и готовая к работе.

## 2. Как устроена оригинальная игра (реконструкция по именам классов)

Игровой цикл оригинала виден по командам (`Backgammon.Gameplay.Commands`) — это
классический паттерн «фазы хода»:

```
StartTurnCommand
  └─ EnterRollDicePhaseCommand → RollDicesCommand → ExitRollDicePhaseCommand
  └─ EnterCheckerMovementPhaseCommand
        → GameplayValidateCheckerCommand   (проверка легальности хода)
        → MakeVisualMoveCommand / MakeForcedMoveCommand
     ExitCheckerMovementPhaseCommand
  └─ EndTurnCommand
SynchronizeBoardStateCommand   (синхронизация с сервером)
UpdateAvailableActionsCommand
OfferDoublingCommand           (куб удвоения)
LayOutCheckersCommand          (начальная расстановка)
ShowNoAvailableMovesNotificationCommand
```

Модель данных (`Backgammon.SharedData`) — что гоняется между клиентом и сервером:
`BoardState`, `CheckerMove`, `CheckerPoint`, `CurrentTurn`, `RolledDice`, `DoublingDie`,
`GameState`, `GameResult`, `MatchState`, `GameTurnCheckerMove`, `MatchPlayer`, …

Сцена/визуал (`Backgammon.Match`): `Board`, `Checker`, `CheckerPoint`, `BearingOffPoint`,
`Cup` (стакан), `DiceAnimationController`, `DraggableObject`, `CameraController`.

Поддерживаемые варианты правил (по интерфейсам-провайдерам в `Backgammon.Match`):
`IDoublingModeProvider` (куб удвоения), `ICrawfordModeProvider` (правило Кроуфорда),
`IJacobyModeProvider` (правило Джекоби), `IDiceModeProvider` (режимы кубиков),
`IPrivateModeProvider`, `IWithSpectatorsProvider` (зрители), `ITimeBankProvider` (банк времени).

> Полная карта классов — в файле `architecture_map.txt`.

## 3. Правила, реализованные в движке (классические короткие нарды)

- **Доска:** 24 пункта. У каждого игрока 15 шашек, стандартная стартовая расстановка.
- **Ход:** бросок 2 кубиков. Одинаковые значения (**дубль**) дают 4 хода этим числом.
- **Перемещение:** каждый кубик = ход одной шашки на это число пунктов в своём направлении
  (белые — к пункту 1, чёрные — к пункту 24).
- **Блокировка:** нельзя встать на пункт, где ≥2 шашек соперника.
- **Взятие («бить блот»):** встав на пункт с ровно 1 шашкой соперника, отправляешь её на **бар**.
- **Бар:** пока есть шашки на баре — обязан сначала ввести их в дом соперника.
- **Вынос («bearing off»):** когда все 15 шашек в своём доме — можно выносить за край;
  кубик, превышающий дистанцию, выносит дальнюю шашку (если нет более дальних).
- **Победа:** первый, кто вынес все 15 шашек.

Что НЕ включено (легко добавляется поверх): куб удвоения, Crawford/Jacoby, «длинные
нарды», строгое правило «использовать оба кубика / больший кубик», бот.

## 4. API движка (`nardy-engine.js`)

Подключение:

```js
// Node:
const Nardy = require('./nardy-engine.js');
// Браузер: <script src="nardy-engine.js"></script>  →  window.Nardy
```

Основные функции:

| Функция | Назначение |
|---|---|
| `initState()` | начальное состояние партии |
| `startTurn(s)` | бросить кубики, заполнить `s.dice`; вернёт `[a,b]` |
| `rollDice()` / `diceToMoves(roll)` | низкоуровневый бросок и разворот дубля в 4 хода |
| `allLegalMoves(s)` | все легальные ходы текущего игрока: `[{from,to,die}]` |
| `legalMovesFrom(s, from)` | легальные ходы из пункта (`from` = индекс или `'bar'`) |
| `applyMove(s, from, to, die)` | применить ход (мутирует `s`), вернёт `{hit}` |
| `hasAnyMove(s)` | есть ли ход (иначе — передача) |
| `endTurn(s)` | передать ход сопернику |
| `pipCount(s, 'w')` | pip-счёт игрока (меньше — лучше) |
| `isGameOver(s)` / `winner(s)` | конец игры / победитель `'w'`/`'b'` |
| `cloneState(s)` | глубокая копия (для перебора/ИИ) |
| `autoPlayTurn(s, pick?)` | заготовка бота: разыграть все кубики |

Состояние `s`:

```js
{
  pts:  Int[24],          // >0 белые, <0 чёрные, 0 пусто
  bar:  { w:Int, b:Int }, // шашки на баре
  off:  { w:Int, b:Int }, // вынесенные (15 = победа)
  turn: 'w' | 'b',
  dice: Int[],            // оставшиеся кубики, напр. [3,5] или [6,6,6,6]
  rolled: [a,b] | null
}
```

Пример полного хода:

```js
const s = Nardy.initState();
Nardy.startTurn(s);                 // напр. s.dice = [3, 5]
const moves = Nardy.allLegalMoves(s);
const m = moves[0];
Nardy.applyMove(s, m.from, m.to, m.die);
if (s.dice.length === 0 || !Nardy.hasAnyMove(s)) Nardy.endTurn(s);
```

Проверка движка: `node nardy-engine.js --selftest` (8 проверок + 200 авто-партий).

## 5. Куда расширять

- **Бот:** замени случайный выбор в `autoPlayTurn` на оценку (минимизировать `pipCount`,
  не оставлять блоты под боем, строить «дома»). Для перебора используй `cloneState`.
- **Куб удвоения:** добавь поле `cube:{value, owner}` и команды offer/accept/reject.
- **Длинные нарды:** другой вектор движения и запрет бить — отдельный режим, ядро то же.
- **Онлайн:** как в оригинале — сервер хранит `BoardState`, валидирует ходы (этим же
  движком на сервере) и рассылает обновления; клиент только рисует и шлёт `CheckerMove`.
