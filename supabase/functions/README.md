# Edge Functions — серверный авторитет правил (Фаза 5)

Развёрнуты в проект A-NARDS (`wiwfzmwzkjmluhnrrcnh`), все с `verify_jwt = true`.

| Функция | Вход | Что делает |
|---|---|---|
| `start-game` | `{ table_id }` | Проверяет два места за столом, создаёт `games` (начальная расстановка, случайный первый ход), переводит стол в `playing`. Идемпотентна — вернёт текущую активную партию. |
| `roll-dice` | `{ game_id }` | Сервер бросает кубики текущего игрока (детерминированный ГСЧ по `seed+ply`). Если ходов нет — авто-пас и смена игрока. |
| `play-move` | `{ game_id, move:{from,to,die} }` | Валидирует ход тем же движком (очередь, легальность, **максимальное использование кубиков**), применяет, пишет в `moves`. При завершении партии — `finalize_game` (статистика + Elo). |

## Авторитет и безопасность

- Записи в `games`/`moves` идут только из функций под **service_role** (RLS у пользователя на запись нет) — защита от читов.
- Личность вызывающего — по его JWT (`requireUser`); цвет берётся из `table_seats`.
- Кубики бросает **сервер**, клиент не присылает значения.

## Изоморфный движок

`_shared/{types,engine,rules}.ts` — Deno-копии клиентского движка, генерируются из
`src/engine` и `src/game/rules.ts` скриптом `scripts/sync-server-engine.mjs`
(паритет клиент↔сервер по построению). После правок движка — перегенерировать и пере-деплоить.

## Повторный деплой

Через Supabase MCP (`deploy_edge_function`) или CLI:
`supabase functions deploy start-game roll-dice play-move`.
В наборе файлов для каждой функции должны быть `index.ts` и общие `../_shared/*.ts`.
