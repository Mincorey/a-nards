| Вопрос | Решение |

| Стек фронтенда | **React + Vite + TypeScript** |

| Деплой | **Vercel** (фронт) + Supabase (бэк) |

## 1. Видение продукта

Веб-игра в нарды, одинаково удобная на десктопе, планшете и смартфоне.

**Что умеет игрок:**
- Зайти без регистрации и сыграть быструю партию с ботом.
- Зарегистрироваться → получить профиль, статистику, рейтинг.
- Играть по сети с другими игроками.
- Создать свой стол (комнату), настроить правила, пригласить друга.
- Подключиться к открытому столу из списка («быстрая игра онлайн»).
- Добавлять других в друзья, видеть их онлайн-статус, звать за стол.

**Ключевые ощущения:** красивая деревянная доска, физика брошенных кубиков (3D-куб из развёртки), плавное перетаскивание фишек пальцем/мышью или тапом, понятная подсветка возможных ходов.

## 2. Архитектура (высокоуровнево)

```
┌────────────────────────────────────────────────────────┐
│                     КЛИЕНТ (браузер)                     │
│  React + Vite + TS                                       │
│  ├─ UI-слой: экраны (меню, лобби, профиль, игра)         │
│  ├─ Игровой рендер: <Board/> (Canvas/SVG) + фишки + кубики│
│  ├─ Движок правил (nardy-engine) — для локальной игры,   │
│  │   предсказания ходов и подсветки (клиентская копия)   │
│  └─ Supabase JS client: Auth, Realtime, RPC              │
└───────────────┬─────────────────────────┬───────────────┘
                │ Realtime (WS)            │ HTTPS (RPC/Edge)
┌───────────────▼─────────────────────────▼───────────────┐
│                    SUPABASE (бэкенд)                       │
│  ├─ Auth (email/пароль, OAuth опц.)                       │
│  ├─ Postgres + RLS (профили, друзья, столы, игры, ходы)   │
│  ├─ Realtime (подписки на состояние стола/игры/чата)      │
│  ├─ Edge Functions (Deno):                               │
│  │    • тот же движок правил → валидация и применение хода│
│  │    • матчмейкинг, старт партии, тайм-банк, итоги       │
│  └─ Storage (аватары; ассеты можно отдать с фронта)      │
└──────────────────────────────────────────────────────────┘
```

**Почему сервер-авторитет:** клиент не должен сам решать, легален ли ход и что выпало на кубиках — иначе тривиальный чит. Бросок кубиков и валидацию ходов в онлайне делает Edge Function. Клиентский движок нужен для офлайн-игры с ботом и для мгновенной подсветки/предсказания (UX), но в онлайне его результат всегда подтверждается сервером.
