### Шаг 3. Storage — аватары (2 файла)

- Самый простой путь: пока не удаляем старый проект, 2 пользователя с аватарами (**Олег** и **СУХУМ**) просто заново загрузят фото на новом сайте — `avatar_url` обновится сам.

### Шаг 4. Edge Functions (6 штук) — деплой + секреты
Деплою все 6 функций в новый проект. Затем в новом проекте (Dashboard → Edge Functions → Secrets, либо CLI) прописываем 3 секрета: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_ADMIN_CHAT_ID`, `ALLOWED_ORIGINS` (в последний — адрес(а) сайта на Vercel). Значения токенов я прочитать не могу — их вставляешь ты.

### Шаг 5. Auth-настройки нового проекта
В Dashboard нового проекта → Authentication → URL Configuration:
- **Site URL** = адрес сайта на Vercel (напр. `https://a-nards.vercel.app`).
- **Redirect URLs** = тот же адрес (+ кастомный домен, если есть).
Проверить, что провайдер **Email** включён (регистрация по email/паролю).

### Шаг 6. Vercel — переменные + пересборка
В проекте на Vercel (Settings → Environment Variables) поменять на значения нового проекта:
- `VITE_SUPABASE_URL` = `https://gmpvcrsmkxpjkjrvuyng.supabase.co`
- `VITE_SUPABASE_ANON_KEY` = (новый anon-ключ, лежит в `_migration_fr/NEW_env.local.txt`)

**Обязательно Redeploy** (Vite «вшивает» эти переменные в бандл на этапе сборки — без пересборки старый ключ останется в коде).

### Шаг 7. Локально
Заменить содержимое `.env.local` на `_migration_fr/NEW_env.local.txt` и обновить `Supabase.txt` новыми ключами (данные нового проекта — в файле `NEW SUPABASE.txt`).

### Шаг 8. Тесты на новом проекте
Регистрация нового пользователя, вход старым логином/паролем, загрузка аватара, создание стола/игры, ход + бросок костей, отправка формы обратной связи (проверить, что сообщение падает в Telegram).

### Шаг 9. Финал
Только после успешных тестов — поставить старый сингапурский проект на паузу (на пару дней для подстраховки), затем удалить. Старые ключи (в старом `Supabase.txt`) считать скомпрометированными.
