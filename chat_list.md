## Сессия 2 — 2026-06-28 — Фаза 3 (анимации и управление) выполнена

**Сделано:**
- **Drag-and-drop + тап** (`Board.tsx`): единый код мышь+тач на Pointer Events.
  pointerdown захватывает легальный источник, поднятая «ghost»-фишка следует за указателем,
  pointerup на легальной цели делает ход, иначе фишка возвращается. Тап-управление сохранено
  тем же обработчиком. client→viewBox-координаты через `getScreenCTM().inverse()`; хит-тест по
  ячейкам пунктов и перемычке (бар/вынос). `touch-action: none`.
- **3D-кубик** (`Dice3D.tsx` + `dice3d.css`): HTML-оверлей над SVG (SVG не умеет preserve-3d).
  Куб из 6 граней с точками; обороты копятся монотонно по `rollId` → кувырок играет и при
  появлении, и при каждом новом броске, с точным приземлением на выпавшую грань.
  Решение: грани с ТОЧКАМИ (а не нарезка ч/б `dice.png`) — чётче и гарантированно верные значения.
  `useGame` теперь отдаёт `rollId` (инкремент на каждый startTurn); `Dice` (2D) больше не используется.
- **Анимации боя/выноса/переходов** (`board.css`): фишки позиционируются через `transform`
  с `transition` → плавная осадка стопки; влёт сбитой фишки на бар (`bd-pop`); появление
  вынесенной фишки (`bd-slide-in`). `prefers-reduced-motion` отключает анимации.
- **Мобильная полировка** (`index.css`): доска во всю ширину, тач-цели ≥44px, `hover:none`.
- **Тесты:** добавлен `Board.interaction.test.tsx` (jsdom + @testing-library) — тап-выбор,
  тап-по-цели, drag-drop, нелегальный drop. Добавлены devDeps: jsdom, @testing-library/react/dom.

подтвердил через SQL-симуляцию роли `authenticated` (rollback-транзакции) до применения фикса.

## Session 5 (2026-07-02) — фикс дублирующейся Realtime-подписки

**Проблема:** после фикса мобильного меню профиля (Session 4, п.1) в консоли на

`Uncaught Error: cannot add postgres_changes callbacks for realtime:social:<id>
after subscribe(). at ProfileMenu.tsx:34:12 ... at subscribeSocial (friends.ts:147:6)`

**Причина:** `NotificationsBell` и `ProfileMenu` теперь ОБА всегда смонтированы
в DOM (просто скрыты через CSS `.hide-mobile` в зависимости от ширины экрана),
и каждый независимо вызывал `subscribeSocial(auth.user.id, reload)` из
`lib/friends.ts`. Эта функция открывает Supabase Realtime-канал с ФИКСИРОВАННЫМ
именем `social:${me}` — два независимых вызова `.subscribe()` на канал с одним
и тем же именем топика вызывают эту ошибку в клиенте Realtime.

**Фикс:** вынесли общее состояние (заявки в друзья + инвайты + сама Realtime-
подписка) в новый контекст `src/lib/socialNotifications.tsx`
(`SocialNotificationsProvider` / `useSocialNotifications()`), подключённый
ОДИН раз в `main.tsx` (внутри `AuthProvider`, снаружи `BotGameSessionProvider`).
`NotificationsBell.tsx` и `ProfileMenu.tsx` теперь оба просто читают
`const { requests, invites, reload } = useSocialNotifications()` вместо того,
чтобы каждый сам дергал `getFriends`/`getIncomingInvites`/`subscribeSocial`.

**Урок на будущее:** если компонент открывает Realtime-подписку с
ФИКСИРОВАННЫМ именем канала (например, привязанным к user id, а не к
случайному/уникальному идентификатору), и этот компонент может быть смонтирован
в нескольких местах приложения одновременно (в т.ч. просто CSS-скрытый, а не
реально размонтированный через condition/route) — подписку нужно поднимать в
общий React Context на уровне выше, а не дублировать в каждом компоненте.
Иначе Supabase Realtime кинет `cannot add postgres_changes callbacks ... after
subscribe()`.

## Session 11 (2026-07-02) — 4 бага со скриншотов: закрытие модалки, невидимый футер, отступ фона, opening-roll на десктопе

1. **Закрытие модалки настройки игры без «Начать игру» всё равно показывало

2. **Футер не отображался ни на одной странице (важный баг из прошлой сессии).**

   скрыть футер только в компактном мобильном ландшафте игры), из-за
   неправильно подобранного маркера вставил эту строку ПОСЛЕ закрывающей
   скобки `}` медиа-блока — то есть правило оказалось на верхнем уровне и
   скрывало футер ГЛОБАЛЬНО на всех страницах и разрешениях экрана. Поймано

Причина, почему не убиралось раньше: `orientation:portrait` в `site.webmanifest` действует ТОЛЬКО для установленного PWA, а не во вкладке браузера. Решение: оверлей «Поверните телефон вертикально» (рендер в `App.tsx`), показывается media-запросом `(orientation:landscape) and (max-height:500px) and (pointer:coarse)`. ВАЖНО: оверлей НЕ показывается на странице активного стола `/table/:id` — там доска специально свёрстана под ландшафт (много ландшафт-CSS игры). Если нужно блокировать и в игре — убрать условие `!onTable` в App.tsx.
   - ✅ Белая системная полоса снизу: у `body` фон задавался через `background:` (сбрасывает `background-color` в прозрачный) → системная панель белела. Добавлен сплошной `background-color:#241710` на `html` и `body`.
